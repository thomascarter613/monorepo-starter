# Architecture Decision Records

This directory contains Architecture Decision Records for `monorepo-starter`.

An Architecture Decision Record documents an important technical, architectural, governance, tooling, workflow, or product-structure decision.

ADRs should be created when a decision is hard to reverse, affects repository structure, changes developer workflow, introduces a major dependency, defines a long-term convention, or establishes project governance.

## ADR Naming

Use the following format:

```text
adr-0001-record-architecture-decisions.md
adr-0002-choose-primary-runtime-and-package-manager.md
adr-0003-choose-monorepo-task-runner.md
```

## ADR Statuses

Allowed statuses:

* Proposed
* Accepted
* Superseded
* Deprecated
* Rejected

## ADR Template

Each ADR should include:

* Title
* Status
* Date
* Context
* Decision
* Consequences
* Alternatives Considered
* Follow-up Work
