## Step 4 — Create the canonical root folder skeleton

Now create the root directory structure from the charter before installing tooling. This makes the repo shape explicit and gives every future tool, package, app, service, policy, and doc a correct home. The charter defines this as a full governance-grade monorepo structure, not a minimal app scaffold. 

From the repo root, run:

```bash
mkdir -p \
  apps \
  services \
  packages \
  libs \
  tools \
  configs \
  contracts \
  infra \
  policies \
  docs \
  tests \
  examples \
  templates \
  profiles \
  scripts \
  .github \
  .devcontainer \
  .moro
```

Add placeholder files so Git tracks the empty directories:

```bash
for dir in \
  apps \
  services \
  packages \
  libs \
  tools \
  configs \
  contracts \
  infra \
  policies \
  tests \
  examples \
  templates \
  profiles \
  scripts \
  .github \
  .devcontainer \
  .moro
do
  touch "$dir/.gitkeep"
done
```

Do **not** add `docs/.gitkeep`, because `docs/architecture/decisions` already exists.

Then commit:

```bash
git add \
  apps \
  services \
  packages \
  libs \
  tools \
  configs \
  contracts \
  infra \
  policies \
  tests \
  examples \
  templates \
  profiles \
  scripts \
  .github \
  .devcontainer \
  .moro

git commit -m "chore(repo): create canonical monorepo folder structure"
git push
```
