## Step 5 — Add the root `.gitignore`

Before installing dependencies or generating lockfiles, create the repository’s ignore policy. This prevents local artifacts, secrets, build outputs, dependency folders, editor files, and generated caches from entering the first real implementation history.

Your charter is Bun-first, Docker-first, local-first, CI-ready, and security-conscious, so the ignore file should cover Node/Bun artifacts, environment files, build outputs, test outputs, Docker/local data, logs, and IDE noise from the beginning. 

From the repo root, run:

```bash
cat > .gitignore <<'EOF'
# Dependencies
node_modules/
.pnp/
.pnp.*
.yarn/*
!.yarn/patches/
!.yarn/plugins/
!.yarn/releases/
!.yarn/sdks/
!.yarn/versions/

# Package manager artifacts
npm-debug.log*
yarn-debug.log*
yarn-error.log*
pnpm-debug.log*
.bun-install.log

# Bun
.bun/

# Environment files
.env
.env.*
!.env.example
!.env.*.example

# Build outputs
dist/
build/
out/
.output/
.next/
.nuxt/
.svelte-kit/
.vite/
.turbo/
.moon/cache/
.moon/docker/
.cache/
tmp/
temp/

# Test and coverage outputs
coverage/
test-results/
playwright-report/
blob-report/
.nyc_output/

# Logs
logs/
*.log
*.pid
*.seed
*.pid.lock

# TypeScript
*.tsbuildinfo

# OS files
.DS_Store
Thumbs.db

# Editor and IDE files
.idea/
.vscode/*
!.vscode/extensions.json
!.vscode/settings.json

# Local databases and storage
*.sqlite
*.sqlite3
*.db
data/local/
data/tmp/
infra/local/data/

# Docker and local runtime state
docker-compose.override.yml
*.local.yml

# Secrets and certificates
*.pem
*.key
*.crt
*.p12
*.pfx

# Generated reports
reports/generated/
reports/tmp/

# Misc
*.bak
*.swp
*.swo
EOF
```

Then commit it:

```bash
git add .gitignore
git commit -m "chore(repo): add root gitignore"
git push
```
