## Step 3 — Create the root project manifest

Now create the root `package.json`.

Do **not** install dependencies yet. This step only establishes the repository as a private Bun-first monorepo workspace. Your charter defines `monorepo-starter` as Bun-first, TypeScript-first, and organized around apps, services, packages, libs, tools, configs, contracts, docs, templates, profiles, and scripts. 

Run this from the repo root:

```bash
BUN_VERSION="$(bun --version)"

cat > package.json <<EOF
{
  "name": "monorepo-starter",
  "version": "0.0.0",
  "private": true,
  "description": "A governance-grade, supply-chain-aware, policy-driven monorepo starter with lifecycle automation and the moro CLI.",
  "license": "MIT",
  "type": "module",
  "packageManager": "bun@$BUN_VERSION",
  "engines": {
    "bun": ">=$BUN_VERSION"
  },
  "workspaces": [
    "apps/*",
    "services/*",
    "packages/*",
    "libs/*",
    "tools/*",
    "configs/*"
  ],
  "scripts": {
    "check": "bun run format:check && bun run lint && bun run typecheck && bun run test",
    "format": "echo 'format not configured yet'",
    "format:check": "echo 'format check not configured yet'",
    "lint": "echo 'lint not configured yet'",
    "typecheck": "echo 'typecheck not configured yet'",
    "test": "echo 'tests not configured yet'",
    "build": "echo 'build not configured yet'",
    "doctor": "echo 'moro doctor not implemented yet'"
  }
}
EOF
```

Then validate and commit:

```bash
bun pm pkg get name
git add package.json
git commit -m "chore(repo): add root package manifest"
git push
```

This gives the repository its first executable project identity without prematurely adding tooling.
