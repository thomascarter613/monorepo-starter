````bash
# ADR-0002: Use Bun as the Default Runtime and Package Manager

## Status

Accepted

## Date

2026-06-26

## Context

`monorepo-starter` needs a default JavaScript/TypeScript runtime and package-manager strategy.

The project charter defines the starter as Bun-first, TypeScript-first, local-first, and automation-friendly. The starter will include applications, services, packages, generators, documentation, quality gates, and the `moro` CLI.

The project also needs to support alternative package-manager profiles, especially pnpm, without making the default repository ambiguous or committing multiple competing lockfiles.

## Decision

`monorepo-starter` will use Bun as the default runtime and package manager.

The canonical default lockfile will be:

```text
bun.lock
```

The repository will not commit multiple package-manager lockfiles for the same generated profile.

pnpm will be supported as an alternative profile, but not as the default v1 package-manager mode.

The default project commands, scripts, documentation, and examples will use Bun unless a specific alternative profile is selected.

## Consequences

This gives the project a fast, modern, TypeScript-friendly default runtime and package-management experience.

It keeps the default developer workflow simple and consistent.

It aligns the `moro` CLI with the same runtime used by the starter itself.

Supporting pnpm as an alternative profile will require profile-specific templates, documentation, lockfile behavior, and validation rules.

The repository doctor should eventually detect conflicting lockfiles and report them as package-manager strategy violations.

## Alternatives Considered

### pnpm as the default

pnpm is mature, widely used, and excellent for monorepos. It remains the primary alternative package-manager profile.

Rejected as the default because the project charter explicitly selects Bun-first behavior and because Bun provides a simpler unified runtime/package-manager/test-runner experience for the initial path.

### npm as the default

Rejected. npm is universal, but it is not the strongest default for this project’s intended monorepo workflow.

### Yarn as the default

Rejected. Yarn has strong capabilities, but it adds more package-manager strategy complexity than this starter needs by default.

### Supporting all package managers equally from day one

Rejected. The project should have one clear golden path first, then profile-based alternatives.

## Follow-up Work

* Add the root `package.json`.
* Add `.gitignore` rules for package-manager artifacts.
* Add `mise.toml` to pin Bun.
* Add initial Bun workspace configuration.
* Add package-manager validation to the future `moro doctor`.
* Document the pnpm profile separately before implementing it.
EOF

````
